# Next steps (kolejnosc prac)

Krotka lista priorytetow na najblizszy czas, zebrana z brakow w `features.md` / `features_en.md`.

1) Struktura plikow: separatory `=====` w pliku + `export ... to "..."!` / `import ...!`.
2) OOP / klasy: `Nazwa is a class { ... }`, singleton per nazwa, `source` jako `this`, indeksowane pola/metody.
3) Stringi: dowolna liczba cudzyslowow (w tym 0) + interpolacje (np. `${name}`, `{name}?`).
4) `previous` / `next` / `current` jako keywordy bez nawiasow + `await next`.
5) Indenty/whitespace: reguly 3 spacji i pelniejsze zachowanie whitespace-math z README.
